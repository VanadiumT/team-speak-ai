# TeamSpeak AI — 系统总览

## 项目定位

TeamSpeak AI 是一个**三层语音桥接系统**，为 TeamSpeak 语音频道注入 AI 处理能力（语音识别 → 大模型推理 → 语音合成），让玩家在游戏中通过语音与 AI 实时对话。

## 服务拓扑

```
TeamSpeak Client ──→ Java Bridge ──→ Python Backend ──→ Frontend (Vue 3)
    (ts3j)            :8080              :8000              :5173
                    WebSocket          WebSocket           HTTP dev server
```

| 层 | 技术栈 | 职责 |
|---|--------|------|
| **Bridge** | Java 17, Tomcat WS, ts3j | 连接 TeamSpeak 服务器，捕获/播放音频 |
| **Backend** | Python 3.11+, FastAPI, Uvicorn | 管线引擎、AI 推理调度、状态管理 |
| **Frontend** | Vue 3, Vite 5, Pinia | 可视化流程编辑器、实时状态渲染 |

## 数据流向

```
TeamSpeak 语音
  → Java Bridge (ts3j 音频捕获)
    → WS :8080/teamspeak/voice
      → Python AudioBus (发布/订阅)
        → STT 识别 → LLM 推理 → TTS 合成
          → WS :8080/teamspeak/voice
            → Java Bridge → TeamSpeak 播放
```

同时，前端通过 `/ws` 实时查看/编辑管线、监控执行状态。

## 核心设计决策

1. **后端持有全部状态** — 节点坐标、连线、配置全在后端，前端只是编辑器和渲染器
2. **一切走 WebSocket** — 不混用 REST，统一信封协议
3. **流式/非流式双通道** — 实时语音场景要求逐片处理，不能等全文
4. **显式数据流** — 流程参数通过专用节点读写，画布上可见可追踪
5. **预设系统** — 参数密集场景下，保存配置方案并快速切换；不是 AI 专属，是通用机制
