# 编辑器设计

> 定义前端可视化流程编辑器的交互模式、布局、面板系统和核心操作。
> 本文档描述**交互设计思想**，不绑定具体节点类型。

---

## 1. 布局哲学

### 1.1 五区域

```
┌─ Top Nav Bar (56px) ──────────────────────────────────────┐
├─ Sidebar (256px) ──┬─ Canvas (flex-1) ──┬─ Right Panel ───┤
│   工作流树          │   2000×1500       │   320px          │
│   节点设置          │   可缩放/平移     │   单击节点打开    │
│   系统设置          │                   │   流程模式只读    │
├─────────────────────┴───────────────────┴─────────────────┤
├─ Bottom Status Bar (32px) ────────────────────────────────┤
└──────────────────────────────────────────────────────────┘
```

### 1.2 设计理念

- **侧栏窄而固定 (256px)** — 流程导航不需要大空间，折叠/展开成本高
- **画布是主角** — 占据剩余全部空间，不设最大宽度
- **面板按需出现 (320px)** — 单击节点才打开，不抢占画布空间
- **底栏常驻** — 服务状态一目了然，不被打断

## 2. 双模式设计

编辑器有两种模式，由顶栏按钮切换：

| 能力 | 流程模式 (默认) | 编辑模式 |
|------|:---:|:---:|
| 查看节点和连线 | ✓ | ✓ |
| 端口圆点 | 仅已连线/有来源 | 全部显示 |
| 拖拽节点 | ✗ | ✓ |
| 工具面板 | ✗ | ✓ |
| 右侧面板 | 只读 | 可编辑 |
| 端口浮动弹窗 | ✗ | ✓ |
| 撤销/重做 | ✗ | ✓ |
| 删除节点/连线 | ✗ | ✓ |

### 2.1 为什么是双模式而非单模式

- **降低误操作** — 流程运行中或查看时不会意外拖歪节点
- **视觉清洁** — 流程模式下隐藏编辑控件，聚焦数据流
- **心理模型清晰** — 查看 vs 编辑是两个不同的任务

### 2.2 模式切换

- 流程模式点 `[流程设置]` → 进入编辑模式
- 编辑模式点 `[流程模式]` → 返回流程模式（自动保存）

## 3. 画布交互

### 3.1 坐标系

- 节点使用**绝对定位**，`(x, y)` 相对画布左上角
- 画布默认 2000×1500px，可扩展到 5000×5000
- 缩放 25%~300%，变换原点 `(0, 0)`

### 3.2 交互模式

| 操作 | 行为 |
|------|------|
| 中键拖拽 | 平移画布 |
| 滚轮 | 缩放 (以鼠标位置为中心) |
| 从工具面板拖拽 | 创建新节点于画布坐标 |
| 从输出端口拖拽 | 创建临时连线 |
| 拖拽节点卡片 | 移动节点位置 |

### 3.3 工具面板

编辑模式下悬浮于画布左侧。节点按类别分组（起点/输入/变量/处理/音频/输出），每项显示图标、名称、端口数（入↘出）：

```
┌─ 起点 ──┐
│ ▶ 开始     0↘2 │
├─ 输入 ──┤
│ 📤 ... │
├─ 变量 ──┤
│ ... │
├─ 处理 ──┤
│ ... │
├─ 音频 ──┤
│ ... │
├─ 输出 ──┤
│ ... │
└──────────┘
```

数据类型由后端 `registry.py` 统一下发，前端动态渲染。

## 4. 连线系统

### 4.1 两种连线

| 类型 | 视觉 | 承载 |
|------|------|------|
| **数据连线** | 绿色虚线 `#4edea3`, dash 10-5 | 数据传递 (image/audio/string...) |
| **事件连线** | 蓝色实线 `#adc7ff` | 控制信号 (触发/完成) |

### 4.2 创建与删除

- 从输出端口拖拽到输入端口创建连线
- 系统自动判断 data_type 为 `event` → 事件连线，其他 → 数据连线
- 点击连线选中后 Delete 删除，或右键菜单 `[删除连线]`

### 4.3 路径算法

- 水平距离短时用 L 形折线
- 水平距离长时用贝塞尔曲线

### 4.4 渲染层

SVG overlay 层，`pointer-events: none`，hover/click 时单独处理命中区域。

## 5. 面板系统

### 5.1 右侧详情面板 (DynamicPanel)

单击节点打开，固定于右侧 320px。两模式均显示，流程模式只读（隐藏编辑 tab）。

**五个 Tab**：

| Tab | 用途 | 流程模式 |
|-----|------|:---:|
| 详情 | 节点状态渲染（等待/处理中/完成/错误/监听） | ✓ |
| 配置 | 节点参数展示（只读 key-value；编辑模式下可修改） | ✗ 隐藏 |
| IO数据 | 各端口上一次运行时的实际输入/输出值 | ✓ |
| IO管理 | on-demand 端口显隐、repeatable 端口实例增减 | ✗ 隐藏 |
| 日志 | 实时日志流 (FIFO 200条)，颜色按级别 | ✓ |

### 5.2 端口浮动弹窗 (PortPopover)

编辑模式下点击端口圆点弹出（z-150）。输入端口可配置数据来源（连线/固定值/流程参数），输出端口可查看下游连线。

## 6. 视图切换

画布左下角三个按钮控制可见内容：

| 视图 | 数据连线 + 数据端口 | 事件连线 + 事件端口 |
|------|:---:|:---:|
| 全部 | 显示 | 显示 |
| 数据流 | 显示 | 隐藏 |
| 事件流 | 隐藏 | 显示 |

---

## 7. 撤销/重做

- 操作记录格式：`{ action, forward, reverse }`
- 覆盖：node.create/delete/move/update_config/rename, port.move, connection.create/delete
- 栈上限 100 条，新操作清空 redo 栈
- 后端 JSONL 持久化，500ms 合并窗口（同一 config 连续修改合并为一条）

---

## 8. 保存机制

| 触发操作 | 策略 |
|---------|------|
| 节点创建/删除/移动 | 即时保存 |
| switch/toggle/select 配置 | 即时保存 |
| 文本输入配置 | 500ms debounce |
| 端口拖动 | dragend 保存 |

顶栏状态指示器：已保存 `cloud_done` (绿) / 保存中 `sync` (橙旋转) / 未保存 `cloud_off` (灰)。
