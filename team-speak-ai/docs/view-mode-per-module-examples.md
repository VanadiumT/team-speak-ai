# 视图模式下各模块显示示例

> 以上传图片、OCR、LLM 三个模块为例。
> 核心逻辑：**流程模式 = 使用流程，编辑模式 = 设计流程结构**。
> 只有带手动操作的节点（如上传图片）在流程模式下可交互，其余节点只可查看。

---

## 模式职责速览

| | 流程模式 | 编辑模式 |
|------|-------|-------|
| **定位** | 使用流程 | 设计流程结构 |
| **input_image：上传文件** | ✓ 可交互（这是"使用"动作） | ✓ 可交互 |
| **OCR / LLM：body 区** | 只读，反映节点运行时状态 | 只读，同上 |
| **运行管线** | ✓ `[运行]` 按钮 | ✗ |
| **双击节点** | 打开面板，全只读 | 打开面板，Config tab 可编辑 |
| **拖拽/增删节点** | ✗ | ✓ |
| **创建/删除连线** | ✗ | ✓ |
| **修改配置** | ✗ | ✓ |
| **撤销/重做** | ✗ | ✓ |

三种视图滤镜（全视图 / 数据流视图 / 事件流视图）在两种模式下均可切换，纯显示滤镜。

---

## 模块 1：上传图片 (input_image)

**标签页**：无。整个 body 就是文件上传交互区。
**端口**：img-out (image) / trigger-out (event)

input_image 是唯一带手动操作的节点——上传文件属于"使用流程"，流程模式下可交互。

### 流程模式

input_image 带手动操作——上传文件属于"使用流程"，body 可交互。

```
空闲时（节点保留上次上传的状态）：      运行中（pipeline 执行中 / 上传中）：

           ┌──────────┐                           ┌──────────┐
           │ ① 上传图片 │                          │ ① 上传图片 │
           │          ● img-out                   │          ● img-out
           │ screenshot.png  ← 上次上传的文件      │ screenshot.png
           │ 2.4MB · PNG                         │ ████████░░ 65% ← 进度
           │ ● 已完成                              │ ● 上传中    ← 蓝色脉冲
           │          ● trigger-out                │          ● trigger-out
           └──────────┘                           └──────────┘

   点击 body → 可重新上传                        双击 → 无反应（无 tab）
 （若从未上传过则显示上传拖拽区）
```

**body 在不同节点状态下的显示**：

| 状态 | body | 边框/动画 |
|------|------|---------|
| 待上传 | 上传拖拽区（可交互） | 灰色 |
| 上传中 | 文件名 + 进度条 | 蓝色脉冲 |
| 已完成 | 文件名 / 大小 / 类型 | 绿色 |
| 失败 | "上传失败" | 红色 |

### 编辑模式

和流程模式显示相同，额外可拖拽移动节点、可删除节点。

## 模块 2：OCR 识别

**标签页**：配置 / 详情 / 日志
**端口**：ocr-in (image) / ocr-out (string)

OCR 没有手动操作——数据到达自动处理。流程模式下只可查看。

### 流程模式

OCR 没有手动操作——body 只读，内容由节点当前运行时状态决定。双击可打开只读面板。

```
空闲时（节点保留上次执行的状态）：      运行中（pipeline 执行中）：

         ┌──────────────┐                  ┌──────────────┐
         │ ② OCR 识别    │                  │ ② OCR 识别    │
     ● ocr-in        ● ocr-out         ● ocr-in        ● ocr-out ← 流动
         │              │                  │              │
         │ "暗区突围 S3  │                  │ 识别中...     │
         │  赛季更新公告 │                  │ ● 处理中      │ ← 蓝色脉冲
         │  将于5月10日…"│                  │ ████████░░ 80% │
         │ ● 已完成      │                  └──────────────┘
         └──────────────┘
                                           运行中的面板实时更新 I/O + 日志
       （若从未执行过则显示"等待图片..."）
```

**body 在不同节点状态下的显示**：

| 状态 | body | 边框/动画 |
|------|------|---------|
| 待命中 | "等待图片..." | 灰色 |
| 处理中 | 识别进度 + 进度条 | 蓝色脉冲 |
| 已完成 | OCR 文本摘要（前 80 字）| 绿色 |
| 失败 | "OCR 失败" | 红色 |

### 流程模式 — 双击后右侧面板（只读）

```
┌─ OCR 识别 ──────────────────┐
│ [配置] [详情] [日志]         │
├──────────────────────────────┤
│ 配置（只读）：               │
│   引擎: easyocr              │ ← 灰色，不可编辑
│   语言: zh                   │
│   阈值: 0.3                  │
│                              │
│ 详情：                       │
│   ── 输入 ──                 │
│   ocr-in: screenshot.png    │
│   ── 输出 ──                 │
│   ocr-out: "暗区突围 S3..." │
│   上次执行: 14:23:05         │
│                              │
│ 日志：                       │
│   [14:23:01] 引擎加载完成    │
│   [14:23:03] 识别完成       │
│                  共 4 条 ───│
│                  [清空日志]  │
└──────────────────────────────┘
```

### 编辑模式 — 额外可做的

```
流程模式能看的编辑模式都能看，额外：
· 拖拽移动节点
· 删除节点
· 配置 tab 可编辑（改引擎/语言/阈值）
· 通过 "+" 露出 on-demand 端口
```

### 编辑模式 — 右侧面板的配置 tab（可编辑）

```
┌─ OCR 识别 ──────────────────┐
│ [配置] [详情] [日志]         │
├──────────────────────────────┤
│ 引擎:      [easyocr ▾]      │ ← 可编辑
│ 识别语言:  [zh] [en] [+]    │
│ 置信度阈值: [0.3 ────●──]   │
│                              │
│            [保存] [取消]      │
└──────────────────────────────┘
```

## 模块 3：LLM 生成

**标签页**：配置 / 详情 / 日志
**端口**：llm-in (messages) / llm-out (string)

LLM 没有手动操作——上游数据到达自动推理。流程模式下只可查看。

### 流程模式

LLM 没有手动操作——body 只读，内容由节点当前运行时状态决定。双击可打开只读面板。

```
空闲时（节点保留上次执行的状态）：      运行中（pipeline 执行中）：

         ┌──────────────┐                  ┌──────────────┐
         │ ⑦ LLM 生成    │                  │ ⑦ LLM 生成    │
     ● llm-in        ● llm-out         ● llm-in        ● llm-out ← 流动
         │              │                  │              │
         │ "A点有敌人，  │                  │ A点有敌人，注 │ ← 流式文本
         │  注意防守。建 │                  │ 意防守。建议从│   逐字出现
         │  议从B点绕后…"│                  │ B点绕后包抄。▌│
         │ ● 已完成      │                  │ ● 处理中      │
         └──────────────┘                  └──────────────┘

       （若从未执行过则显示"待触发"）       运行中的面板实时更新流式文本 + 日志
```

**body 在不同节点状态下的显示**：

| 状态 | body | 边框/动画 |
|------|------|---------|
| 待命中 | "待触发" | 灰色 |
| 处理中 | 流式文本逐字刷新 + 光标闪烁 | 蓝色脉冲 |
| 已完成 | 回复摘要（前 80 字） | 绿色 |
| 失败 | "LLM 调用失败" | 红色 |

## 总结

| 操作 | 流程模式 | 编辑模式 |
|------|--------|---------|
| **上传图片** (input_image) | ✓ 可交互 | ✓ 可交互 |
| **查看 body 状态** (OCR/LLM) | ✓ 反映节点状态 | ✓ 同上 |
| **双击看详情/日志** | ✓ 只读面板 | ✓ 面板，config 可编辑 |
| **点击端口** | ✓ 只读弹窗 | ✓ 配置弹窗 |
| **切换视图滤镜** | ✓ | ✓ |
| **运行管线** | ✓ | ✗ |
| **拖拽/增删节点** | ✗ | ✓ |
| **修改配置** | ✗ | ✓ |
| **撤销/重做** | ✗ | ✓ |

> 待命中、处理中、已完成这些**不是独立的视图模式**——它们只是流程模式下节点的运行时状态。pipeline 执行中节点状态实时变化，执行完成后节点保留最后状态（已完成/失败）。
